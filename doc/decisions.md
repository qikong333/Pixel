# Decisions

## 2026-04-23

- 将 `pyxelate` 作为项目中的独立第 5 种像素化算法接入，而不是单独保留试验脚本。
- `generate.py` 中将项目目录、输入目录、输出目录和首页路径提升为模块常量，方便测试替换。
- `pyxelate` 使用 `Pyx(width=..., height=..., palette=num_colors, dither="none")`，以保持和现有“分色数量”参数的语义一致。
- 缺少 `pyxelate` 时采用运行期报错并跳过该算法的策略，避免拖垮其余输出。

## 2026-05-07

- 前端流程从“预置样例图 + GET 参数生成”切换为“单图上传 + POST 生成”。
- 首页初始态不展示默认结果，只有上传与参数区。
- 宽高由浏览器在用户选择图片后读取原图尺寸自动填充，像素格和分色使用推荐默认值 `1 / 1 / 5`。
- 后续改为纯内存一次性生成，移除 `source_image` 复用策略，避免保存上传图和结果文件。
- 为避免页面响应过大，结果页进一步改成“只内嵌缩略图 + `/media/...` 按需加载大图”的结构。
- SVG 默认关闭，不再作为首屏结果的一部分返回。
- 大图会保存在服务进程内存中；页面刷新或离开时主动删除当前会话，短时 TTL 只作为异常退出时的兜底。
